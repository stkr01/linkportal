Why LinkPortal

Knowledge about your tools shouldn’t live in someone’s head.

Silos

Only one team knows where a platform is managed. When they’re away, everyone’s stuck.

Wasted time

Minutes lost every day hunting through bookmarks, chats and wikis for the right link.

Bus-factor risk

Staff changes mean lost context. Critical access knowledge walks out the door.

LinkPortal makes every tool findable, owned and traceable — in one place the whole team can trust.

How it works

From scattered bookmarks to one source of truth.

  1. 1

    Organize

    An admin sets up the category tree. Editors add links with descriptions, owning team, tags and environment.

  2. 2

    Find

    Anyone searches or browses the tree. Favorites and the command palette make the daily links one keystroke away.

  3. 3

    Launch

    Click to open — web apps in a new tab, or rdp:// / ssh:// straight into your desktop client.

Security by design

Built to be trusted with your internal map.

Hashed passwords

bcrypt (cost 12). No plaintext passwords, ever.

HTTP-only JWT

Server-side RBAC

Permissions enforced on the API, not just hidden in the UI.

FAQ

Good questions, quick answers.

Does LinkPortal store passwords or credentials?

No. It deliberately stores only links and metadata. Think of it as a map to your tools, not a vault for secrets.

What do I need to run it?

Node.js for the API and web app, and a single SQLite file for storage. It deploys on Ubuntu behind nginx with systemd — a full guide ships in the repo.

How does the health-check work?

A scheduler periodically checks each link: HTTP(S) via a HEAD request, and rdp/ssh via a TCP port check. You get a status dot, the last successful time, optional per-link extra monitoring, and alerts when a link goes from up to down.

Can I open Remote Desktop or SSH from a link?

Yes. Store an rdp:// or ssh:// URL and clicking it hands off to your OS protocol handler. A one-time per-PC setup (or GPO/Intune rollout) registers the handlers.

Which languages are supported?

Eight: English, Swedish, Spanish, Slovenian, German, Norwegian, Danish and Portuguese — selectable per user.
